Prefix Sum + HashMap
The most-tested pattern in your prep. Master the variants.
1. Subarray Sum Equals K
LC-560

Count subarrays whose sum equals exactly K.

Key Idea: If prefix[j] - prefix[i] = K, then subarray [i+1..j] sums to K.
So check: mp.count(sum - k)
  • Initialize mp[0] = 1 (empty prefix)
  • For each element, add to running sum
  • If sum - k exists in map, add its count
  • Store mp[sum]++
unordered_map<int,int> mp; mp[0] = 1; int sum = 0, count = 0; for (int x : nums) { sum += x; if (mp.count(sum - k)) count += mp[sum - k]; mp[sum]++; } return count;
Time: O(n) Space: O(n)
2. Subarray Sum Divisible by K
LC-974

Count subarrays whose sum is divisible by K.

Key Idea: Two prefix sums with the same remainder mod K → their difference is divisible by K.
Store rem = ((sum%k) + k) % k — the +k handles negative numbers!
⚠️ Always use ((sum%k)+k)%k, never just sum%k — negative modulo is different in C++!
mp[0] = 1; int sum = 0, count = 0; for (int x : nums) { sum += x; int rem = ((sum % k) + k) % k; if (mp.count(rem)) count += mp[rem]; mp[rem]++; } return count;
Time: O(n) Space: O(k)
3. Subarray Div by K (min length 2)
Twist!

Same as above but subarrays must have length ≥ 2.

Key Twist: Delay inserting the first prefix-remainder into the map.
Process nums[0] and compute its remainder, but don't add it to mp yet. Start the loop from index 1. This ensures we never pick single-element subarrays.
int s = 0, c = 0; s += nums[0]; int r = ((s%k)+k)%k; mp[r] = 1; // seed after first element for(int i=1; i<n; ++i){ s += nums[i]; r = ((s%k)+k)%k; if(mp.count(r)) c += mp[r]; mp[r]++; }
4. Twist: Max Non-Overlapping Subarrays (sum=K)
Variant

Count maximum number of non-overlapping subarrays with sum = K.

Greedy Trick: When you find a valid subarray, clear the map and reset sum to 0. This greedily ends the current subarray and starts fresh.
mp[0] = 1; int sum=0, c=0; for(int x : nums){ sum += x; if(mp.count(sum-k)){ c++; mp.clear(); // ← key line! mp[0] = 1; sum = 0; } else mp[sum]++; }
5. LC-1010: Pairs of Songs (% 60)
LC-1010

Count pairs where (time[i] + time[j]) % 60 == 0.

Like Two Sum but mod 60. For each song with duration t, you want complement = (600 - t) % 60.
Use 600 (not 60) to avoid negative modulo issues.
for(int t : time){ int c = 600 - t; count += mp[c%60]; mp[t%60]++; }
⚠️ Why 600? Because -1 % 60 = -1 in C++, but (600-1) % 60 = 59. Always offset to avoid negatives.
6. Two Sum
LC-1

Return indices of pair summing to target.

Idea: For each element, check if its complement (target - nums[i]) is already in the map. If yes → found the pair. Otherwise store current element's index.
unordered_map<int,int> mp; for (int i = 0; i < n; ++i) { int complement = target - nums[i]; if (mp.count(complement)) return {mp[complement], i}; mp[nums[i]] = i; } return {};
Time: O(n) Space: O(n)

Longest Substring Without Repeating Chars
LC-3

Find length of longest substring with all distinct characters.

  • Use HashMap to store last seen index of each char
  • Maintain left pointer l
  • On duplicate: move l = max(l, mp[s[r]] + 1)
  • Update answer: ml = max(r - l + 1, ml)
Why max(l, ...)? To avoid moving the left pointer backwards. The map might have a stale entry from before our current window.
int l = 0, ml = 0; unordered_map<char,int> mp; for (int r = 0; r < s.size(); r++) { if (mp.find(s[r]) != mp.end()) l = max(mp[s[r]] + 1, l); mp[s[r]] = r; ml = max(r - l + 1, ml); } return ml;
Time: O(n) Space: O(26) = O(1)
Longest Repeating Character Replacement
LC-424

Replace at most K characters to make longest substring of one char.

Key Formula: Window is valid when (window_size - maxFreq) ≤ K.
Track max frequency in window. If invalid, shrink from left (but don't update maxFreq on shrink — it's a classic trick to avoid O(26) recalc).
int freq[26] = {}, maxFreq = 0, ans = 0, l = 0; for (int r = 0; r < s.size(); r++) { maxFreq = max(maxFreq, ++freq[s[r]-'A']); while ((r - l + 1) - maxFreq > k) freq[s[l++]-'A']--; ans = max(ans, r - l + 1); } return ans;
Time: O(n) Space: O(26)
Subarrays with K Distinct Elements
LC-992

Count subarrays with exactly K distinct elements.

The Magic Formula:
exactly(K) = atMost(K) − atMost(K−1)

Direct counting of "exactly K" is tricky. But "at most K" is easy with sliding window!
  • Write atMost(nums, k) function
  • Expand right: add element to freq map
  • Shrink left while distinct elements > k
  • Add r - l + 1 to answer (all windows ending at r)
  • Return atMost(k) - atMost(k-1)
int atMost(vector<int>& nums, int k) { int l = 0, ans = 0; unordered_map<int,int> mp; for (int r = 0; r < nums.size(); r++) { mp[nums[r]]++; while (mp.size() > k) { mp[nums[l]]--; if (mp[nums[l]] == 0) mp.erase(nums[l]); l++; } ans += (r - l + 1); } return ans; } return atMost(nums, k) - atMost(nums, k-1);
Time: O(n) Space: O(k)
Valid Palindrome II
LC-680

Can you make a palindrome by deleting at most one character?

  • Two pointers: l=0, r=n-1
  • Move both inward while chars match
  • On mismatch: try skipping s[l] OR s[r]
  • Check if either remaining portion is a palindrome
auto isPalin = [&](int l, int r) { while (l < r) if (s[l++] != s[r--]) return false; return true; }; int l = 0, r = s.size() - 1; while (l < r) { if (s[l] != s[r]) return isPalin(l+1,r) || isPalin(l,r-1); l++; r--; } return true;
Time: O(n) Space: O(1)
Jump Game I
LC-55

Can you reach the last index? Each element = max jump from that position.

Track maxReach. For each index i ≤ maxReach, update maxReach = max(maxReach, i + nums[i]). If maxReach ≥ n-1 at any point → true.
int maxReach = 0; for (int i = 0; i < n; i++) { if (i > maxReach) return false; maxReach = max(maxReach, i + nums[i]); } return true;
Jump Game II
LC-45

Minimum jumps to reach the last index.

Greedy BFS-like approach. Track curEnd (end of current jump range) and farthest (furthest reachable). When i reaches curEnd, increment jumps and set curEnd = farthest.
int jumps = 0, curEnd = 0, farthest = 0; for (int i = 0; i < n - 1; i++) { farthest = max(farthest, i + nums[i]); if (i == curEnd) { jumps++; curEnd = farthest; } } return jumps;
Time: O(n) Space: O(1)

Next Greater Element I
LC-496

For each element in nums1, find its next greater in nums2.

  • Process nums2 with a monotonic stack
  • While stack top < current: it has found its next greater
  • Store in HashMap: mp[popped] = current
  • Look up nums1 elements in the map
stack<int> st; unordered_map<int,int> mp; for (int x : nums2) { while (!st.empty() && st.top() < x) { mp[st.top()] = x; st.pop(); } st.push(x); } vector<int> ans; for (int x : nums1) ans.push_back(mp.count(x) ? mp[x] : -1); return ans;
Time: O(n+m) Space: O(n)
Next Greater Element II (Circular)
LC-503

Circular array — after the last element, wrap around to find the next greater.

Trick: Iterate 2×n times using i % n to simulate the wrap. Push indices (not values) onto the stack so we can correctly map results back. Only push during the first pass (i < n) to avoid duplicating result entries. Traverse right-to-left so the stack already holds "future" candidates when we process each index.
vector<int> nextGreaterElements(vector<int>& arr) { int n = arr.size(); vector<int> ans(n, -1); stack<int> st; // stores VALUES (monotone decreasing) for (int i = 2*n - 1; i >= 0; i--) { int idx = i % n; // Pop elements that are not greater than current while (!st.empty() && arr[idx] >= st.top()) st.pop(); // Only record answer on first pass (i < n) if (i < n && !st.empty()) ans[idx] = st.top(); st.push(arr[idx]); } return ans; }
⚠️ Push values not indices here (unlike Sliding Window Max). The stack holds candidates for "next greater" — we compare values, not positions. Only write ans[idx] when i < n to avoid overwriting with stale second-pass data.
Time: O(n) Space: O(n)
Valid Parenthesis
LC-20

Check if brackets are balanced.

  • Push open brackets onto stack
  • On closing bracket: check top matches
  • If mismatch or stack empty → false
  • At end, stack must be empty
stack<char> st; for (char c : s) { if (c=='('||c=='['||c=='{'){ st.push(c); continue; } if (st.empty()) return false; char top = st.top(); st.pop(); if (c==')'&&top!='(') return false; if (c==']'&&top!='[') return false; if (c=='}'&&top!='{') return false; } return st.empty();
Furthest Building (Greedy + Min-Heap)
LC-1642

Use ladders on the biggest climbs, bricks on the rest.

Greedy + Min-Heap: Use a ladder for every positive jump. Keep all used jumps in a min-heap of size ≤ ladders. When heap exceeds ladders, swap the smallest heap jump to bricks. If bricks run out → return current index.
priority_queue<int,vector<int>,greater<int>> pq; for (int i = 0; i < n-1; i++) { int diff = heights[i+1] - heights[i]; if (diff <= 0) continue; pq.push(diff); if (pq.size() > ladders) { bricks -= pq.top(); pq.pop(); if (bricks < 0) return i; } } return n - 1;
Time: O(n log L) Space: O(L)
▸ BINARY SEARCH VARIANT

Binary search on the answer index: for a candidate mid, collect all positive climbs in [0, mid), sort descending, greedily assign ladders to the largest climbs and bricks to the rest. If feasible, try going further.

int l = 0, h = heights.size() - 1, ans = 0; while (l <= h) { int mid = l + (h - l) / 2; vector<int> climbs; for (int i = 0; i < mid; ++i) if (heights[i+1] > heights[i]) climbs.push_back(heights[i+1] - heights[i]); sort(climbs.begin(), climbs.end(), greater<int>()); int usedBricks = 0, usedLadders = 0; bool possible = true; for (int c : climbs) { if (usedLadders < ladders) usedLadders++; else if (usedBricks + c <= bricks) usedBricks += c; else { possible = false; break; } } if (possible) { ans = mid; l = mid + 1; } else { h = mid - 1; } } return ans;
Greedy insight inside canReach: always use ladders on the largest climbs (sorted desc), so remaining climbs consume the fewest bricks.
Time: O(n² log n) worst Space: O(n)
Gas Station
LC-134

Find starting gas station to complete a circular route.

Two-pass Greedy: If total gas ≥ total cost, a solution always exists. Track running tank; when it goes negative, reset start to i+1 and clear tank. The final start is the answer.
int totalGas = 0, tank = 0, start = 0; for (int i = 0; i < n; i++) { totalGas += gas[i] - cost[i]; tank += gas[i] - cost[i]; if (tank < 0) { start = i+1; tank = 0; } } return totalGas >= 0 ? start : -1;
Time: O(n) Space: O(1)

Number of Islands
LC-200

Count connected components of '1's in a 2D grid.

  • Iterate grid — on finding unvisited '1', increment count
  • BFS: push cell into queue, mark visited immediately on enqueue
  • Expand 4 directions, skip out-of-bounds / water / already visited
  • All cells reachable from this '1' get marked — they belong to the same island
Mark visited on enqueue, not dequeue! If you mark on dequeue, the same cell can be pushed multiple times, causing incorrect counts and TLE on dense grids. Use a separate vis matrix so the original grid is untouched.
int numIslands(vector<vector<char>>& grid) { int n = grid.size(), m = grid[0].size(); vector<vector<int>> dir = {{0,1},{0,-1},{1,0},{-1,0}}; vector<vector<int>> vis(n, vector<int>(m, 0)); int c = 0; for (int i = 0; i < n; i++) { for (int j = 0; j < m; j++) { if (vis[i][j] || grid[i][j] == '0') continue; c++; queue<pair<int,int>> q; q.push({i, j}); vis[i][j] = 1; // mark on ENQUEUE while (!q.empty()) { auto [x, y] = q.front(); q.pop(); for (auto d : dir) { int nx = x + d[0], ny = y + d[1]; if (nx >= 0 && nx < n && ny >= 0 && ny < m && !vis[nx][ny] && grid[nx][ny] == '1') { vis[nx][ny] = 1; // mark on ENQUEUE q.push({nx, ny}); } } } } } return c; }
⚠️ Boundary check: nx >= 0 && nx < n && ny >= 0 && ny < m — always check both axes. Your original code had nx<n && nx>=0 which works but is less readable; keep it consistent.
Time: O(n×m) Space: O(n×m)
Rotten Oranges
LC-994

Minimum time for all oranges to rot. Multi-source BFS.

  • Add all rotten oranges (2s) to queue at once
  • Count fresh oranges
  • BFS level-by-level (each level = 1 minute)
  • If fresh = 0 → return minutes, else -1
Why multi-source? All rotten oranges rot simultaneously, not sequentially. Starting all of them in the queue simulates parallel spreading.
queue<pair<int,int>> q; int fresh=0, mins=0; for(int i=0;i<n;i++) for(int j=0;j<m;j++){ if(grid[i][j]==2) q.push({i,j}); if(grid[i][j]==1) fresh++; } while(!q.empty() && fresh){ int sz=q.size(); mins++; while(sz--){ auto [r,c]=q.front(); q.pop(); for(auto& d:dirs){ int nr=r+d[0],nc=c+d[1]; if(nr>=0&&nr<n&&nc>=0&&nc<m&&grid[nr][nc]==1) { grid[nr][nc]=2; fresh--; q.push({nr,nc}); } } } } return fresh==0 ? mins : -1;
Rotten Oranges — Circular Grid Twist
Twist!

Grid wraps around — last column touches first column, last row touches first row.

// Instead of boundary check: int nx = (x + d[0] + n) % n; int ny = (y + d[1] + m) % m; // +n/+m before %n/%m ensures no negative mod
The boundary check if(nx>=0 && nx<n) becomes redundant — modular arithmetic handles wrapping. But keep the freshness check!

Middle of Linked List
LC-876

Find the middle node. If even length, return the second middle.

Fast/Slow Pointers: slow moves 1 step, fast moves 2 steps. When fast reaches end, slow is at the middle.
while(fast && fast->next){ slow = slow->next; fast = fast->next->next; }
Cycle Detection (Floyd's)
LC-141

Does the linked list have a cycle?

  • slow = fast = head
  • Move slow by 1, fast by 2
  • If they meet → cycle exists
  • If fast reaches null → no cycle
ListNode* slow=head, *fast=head; while(fast && fast->next){ slow=slow->next; fast=fast->next->next; if(slow==fast) return true; } return false;
Cycle II — Entry Point
LC-142

Find where the cycle begins.

After slow and fast meet inside the cycle: reset one pointer to head. Move both one step at a time. Where they meet again = cycle entry point.

Math: Distance from head to entry = distance from meeting point to entry (going forward in cycle).
ListNode* slow=head, *fast=head; while(fast && fast->next){ slow=slow->next; fast=fast->next->next; if(slow==fast){ ListNode* prev=head; while(prev!=slow){ prev=prev->next; slow=slow->next; } return prev; } } return nullptr;

Koko Eating Bananas
LC-875

Find minimum eating rate K to finish all piles in H hours.

Binary search on K: K ranges from 1 to max(piles). For a given K, time = Σ ceil(pile/K). If time ≤ H, K is valid — try smaller. Else try larger.
  • lo=1, hi=max(piles)
  • Compute mid = (lo+hi)/2
  • Calculate total time: Σ (p + mid - 1) / mid
  • If time ≤ h: ans = mid, hi = mid-1
  • Else: lo = mid + 1
⚠️ (p + mid - 1) / mid is integer ceiling division — equivalent to ceil(p / mid).
int lo=1, hi=*max_element(piles.begin(),piles.end()), ans=0; while(lo<=hi){ int mid=lo+(hi-lo)/2, t=0; for(int p:piles) t+=(p+mid-1)/mid; if(t<=h){ ans=mid; hi=mid-1; } else { lo=mid+1; } } return ans;
Time: O(n log(max)) Space: O(1)
Binary Search Template
Template

General pattern for "find minimum valid value":

int lo = 1, hi = maxVal, ans = -1; while(lo <= hi){ int mid = lo + (hi-lo)/2; // avoid overflow if(isValid(mid)){ ans = mid; hi = mid - 1; // try to go lower } else { lo = mid + 1; // need bigger value } } return ans;
Remember: Use lo + (hi-lo)/2 over (lo+hi)/2 to prevent integer overflow for large values.

🔑 Actual JPMC Question: Secure Password

Given a binary string, split it into non-overlapping substrings where each substring has even length and contains only one type of character (all 0s or all 1s). Find the minimum flips to make this possible.

Key Insight: Group consecutive identical characters into runs. Each run of odd length needs exactly 1 flip (to make it even). Scan with two pointers, find each run's length, check parity.
int flips = 0; for(int i=0; i<s.size(); ){ int j = i; while(j < s.size() && s[j]==s[i]) j++; int len = j - i; if(len % 2 == 1) flips++; i = j; } return flips;
Time: O(n) Space: O(1)
Variant: Binary String Partition (diff ≤ K)
Variant

Partition into minimum substrings where |count(1s) - count(0s)| ≤ K.

Greedy: Scan left to right, counting 1s and 0s. When the difference exceeds K, start a new partition. Handle the last partition after the loop with if(ones > 0 || zeroes > 0) partitions++.
int ones=0, zeros=0, partitions=0; for(int i=0; i<n; i++){ if(s[i]=='1') ones++; else zeros++; if(abs(ones-zeros) > k){ partitions++; ones = zeros = 0; i--; // reprocess current char in new partition } } if(ones || zeros) partitions++;

🔋 Partition Array — Sum = Power of Two

Partition the entire array into contiguous subarrays such that the sum of each subarray is a power of 2 (1, 2, 4, 8, 16…). Return true if possible.

Key Insight: This is a reachability problem. Think of each index as a node. You can "jump" from index i to j+1 if sum(i..j) is a power of 2. Ask: can you reach index N?

BRUTE FORCE — O(2^N)

bool isPow2(long long n){ return (n>0) && !(n&(n-1)); } bool solve(int s, vector<int>& a){ if(s==a.size()) return true; long long sum=0; for(int j=s;j<a.size();j++){ sum+=a[j]; if(isPow2(sum) && solve(j+1,a)) return true; } return false; }

OPTIMAL — O(N²) MEMOIZATION

int memo[2001]; // init -1 bool solve(int s, vector<int>& a){ if(s==a.size()) return true; if(memo[s]!=-1) return memo[s]; long long sum=0; for(int j=s;j<a.size();j++){ sum+=a[j]; if(isPow2(sum) && solve(j+1,a)) return memo[s]=1; } return memo[s]=0; }
Time: O(N²) Space: O(N)
⚠️ Use long long for currentSum always. Zeros can cause infinite loops — your index must always advance. If N ≤ 20: raw recursion fine. If N ≤ 2000: use memo. If N ≥ 10⁵: need greedy/O(N log N).
0-1 BFS on Grids
Deque BFS

Shortest path where edges have weight 0 or 1 — e.g., "minimum flips to make a path valid".

Rule: weight=0 → push_front. weight=1 → push_back. This keeps the deque sorted by cost without a heap.
// init dist matrix with INT_MAX deque<pair<int,int>> dq; dq.push_front({0,0}); dist[0][0]=0; while(!dq.empty()){ auto [r,c] = dq.front(); dq.pop_front(); for(auto& d : dirs){ int nr=r+d.first, nc=c+d.second; int w = (grid[r][c]==need)?0:1; if(valid && dist[r][c]+w < dist[nr][nc]){ dist[nr][nc] = dist[r][c]+w; if(w==0) dq.push_front({nr,nc}); else dq.push_back({nr,nc}); } } }
⚠️ Always init dist to INT_MAX. For circular grids use (r+dr+rows)%rows.
Time: O(V+E) vs Dijkstra O(E log V)
Binary Search — Split Array Largest Sum
LC-410

Partition array into K subarrays to minimize the maximum subarray sum.

Same as Koko Bananas pattern — binary search on the answer (the max allowed sum), then greedily check if K splits are enough.
bool canSplit(vector<int>& a, int mid, int k){ int cnt=1, cur=0; for(int x:a){ if(cur+x>mid){ cnt++; cur=x; } else cur+=x; } return cnt<=k; } // lo = max_element, hi = total sum while(lo<=hi){ int mid=lo+(hi-lo)/2; if(canSplit(a,mid,k)){ans=mid;hi=mid-1;} else lo=mid+1; }
⚠️ lo MUST start at max_element — not 0. If a single element > mid, canSplit breaks.
Time: O(N log S) Space: O(1)
Sliding Window Maximum
LC-239

Find the maximum in every sliding window of size K — in O(N).

Monotonic Deque: Store indices (not values!) in a deque. Keep it decreasing. Pop from front when out of window, pop from back when smaller than current.
deque<int> dq; // stores indices for(int i=0;i<n;i++){ // remove out-of-window if(!dq.empty() && dq.front()==i-k) dq.pop_front(); // remove smaller elements while(!dq.empty() && nums[dq.back()]<nums[i]) dq.pop_back(); dq.push_back(i); if(i>=k-1) res.push_back(nums[dq.front()]); }
⚠️ Store indices not values — it's the only way to check i - k (out-of-window eviction).
Time: O(N) vs PQ: O(N log K)
Maximum XOR (Bitwise Trie)
Advanced

Find two numbers in an array whose XOR is maximum.

Trie of bits: Insert all numbers bit-by-bit (MSB first). For each number, greedily pick the opposite bit at every level to maximize XOR. If the opposite child doesn't exist, take what's available.
struct Node { Node* links[2]; }; void insert(int num, Node* root){ Node* cur = root; for(int i=31;i>=0;i--){ int b=(num>>i)&1; if(!cur->links[b]) cur->links[b]=new Node(); cur=cur->links[b]; } } // During query: try (bit ^ 1) first to maximize XOR
⚠️ For positive-only numbers, start from bit 30 (not 31) to skip the sign bit and save iterations.
Time: O(N × 32) Space: O(N × 32)
